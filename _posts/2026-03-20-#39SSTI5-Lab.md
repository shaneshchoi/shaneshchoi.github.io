# Lab: Server-side template injection with information disclosure via user-supplied objects

Lab Link: [Server-side template injection with information disclosure via user-supplied objects](https://portswigger.net/web-security/server-side-template-injection/exploiting/lab-server-side-template-injection-with-information-disclosure-via-user-supplied-objects)

---

## 개요

이번 lab은 **템플릿에서 접근 가능한 객체를 이용해서 정보를 빼오는 케이스**다.

주어진 `content-manager:C0nt3ntM4n4g3r` 계정으로 로그인해서  
웹 애플리케이션 내부의 기능을 활용하여 Framework의 `Secret Key`를 가져오는게 목표이다.

## 4. settings 객체 확인

<div class="img-left-block">
<img src="/assets/burp_images/SSTI/SSTI_lab5_5.png" alt="설명" width="600">
</div>

settings 안에는 여러 값이 들어있는데,  
`ADMINS`를 예로 들어보자면,  

<div class="img-left-block">
<img src="/assets/burp_images/SSTI/SSTI_lab5_6.png" alt="설명" width="1000">
</div>

아래와 같이 Default 값이 [], 즉 빈 문자열 인걸 확인할 수 있는데,  
실제로 `{% raw %} {{settings.ADMINS}} {% endraw%}`를 요청해도  
빈 문자열 []을 리턴하는것 볼 수 있다.

<div class="img-left-block">
<img src="/assets/burp_images/SSTI/SSTI_lab5_7.png" alt="설명" width="600">
</div>

이로서 settings와 관련된 값을들 지정할 수 있다는 취약점이 드러난다.

<div class="img-left-block">
<img src="/assets/burp_images/SSTI/SSTI_lab5_8.png" alt="설명" width="1000">
</div>

우리가 목표로 하는 `SECRET_KEY` 에 대한 documentaiton이 되어 있는데,
이건 Django에서 중요한 보안 값이다. 세션, 암호화 등에 사용되는 값이라고 한다.
`{% raw %}{{settings.SECRET_KEY}}{% endraw %}`로 해당 값을 요청 해볼수 있다.

<div class="img-left-block">
<img src="/assets/burp_images/SSTI/SSTI_lab5_9.png" alt="설명" width="700">
</div>

공격 결과, 화면에 SECRET_KEY 값이 출력되는걸 볼 수 있다..

이 값을 제출하면 lab이 완료된다.
